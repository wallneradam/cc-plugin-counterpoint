description: Reset the counterpoint thread — start a fresh Codex session on the next call
allowed-tools: mcp__counterpoint__reset
---

Reset the counterpoint thread by calling the `mcp__counterpoint__reset` MCP tool.

**The persistent Codex thread has been cleared.** The next `/counterpoint` or `/consult` call will start a brand new Codex session with no memory of prior rounds. Use this when the current thread is stale, holds wrong project context, or when you want to start a fresh debate from scratch.
